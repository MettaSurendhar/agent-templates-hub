# Contributing to the Catalog

This repo is an index, not a codebase — most contributions fall into one of these:

## Adding your template to an existing family

Open an issue using the **"Add a template to the catalog"** template, or skip
straight to a PR adding a row to the relevant table if you're comfortable with the format.

## Proposing a new family

Open an issue using the **"Propose a new family"** template first — let's agree on
the tier breakdown and use case before a table gets added, so the catalog stays coherent.

## Fixing a broken link or stale description

PRs welcome directly, no issue needed.

## Bugs or feature requests on an individual template

Open the issue in _that template's own repo_, not here — this repo only tracks the catalog itself.

## Style guide for table entries

- One-line descriptions should follow the existing pattern: what it does, key stack
  pieces, and what makes this tier distinct from neighboring tiers.
- Keep "Use it when..." to a single sentence — the linked repo's README is where the
  full nuance lives.
